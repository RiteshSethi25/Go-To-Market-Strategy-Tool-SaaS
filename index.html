1. Product complexity5
1–2Single-feature tool, plug-and-play, no workflow change needed
3–4Multi-feature, some setup, light onboarding required
5–6Cross-department, configuration needed, admin training required
7–8Deep integrations, data migration, process re-engineering
9–10Mission-critical, multi-system, 6–18 month implementation
2. Buyer complexity5
1–2Single user buys for themselves, credit card checkout
3–4Team lead or manager decides, 1–2 approvals
5–6Department head + IT sign-off, 3–5 stakeholders
7–8VP/Director + procurement + legal, 6–10 stakeholders
9–10C-suite committee + board approval, 10+ stakeholders, 12+ month cycle
3. Deployment burden5
1–2Live in hours — click, sign up, use immediately
3–41–4 weeks — basic setup, data import, team training
5–61–3 months — configuration, integrations, user onboarding
7–83–6 months — custom workflows, data migration, IT involvement
9–106–18 months — full implementation program, dedicated project team
4. Trust requirements5
1–2Transactional — low switching cost, easy to cancel and replace
3–4Moderate — some data sensitivity, vendor reputation matters
5–6Significant — handles sensitive business data, SOC2 required
7–8High — regulated data, compliance audits, executive sponsorship needed
9–10Institutional — mission-critical, regulatory mandated, reputational risk if it fails
5. PLG / viral viability5
1–2No self-serve path — every deal requires a sales conversation
3–4Limited trial possible but conversion needs sales assist
5–6Free tier exists, some self-serve, sales needed for enterprise
7–8Strong free-to-paid funnel, organic word-of-mouth, team virality
9–10Naturally viral — each user invites others, bottom-up enterprise adoption
